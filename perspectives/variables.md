# Variables

変数設計に関する品質レンズ。

## この観点で重視すること

- **Immutable by default**: `const` / `val` / `let` (Rust) をデフォルトにする。`let` (JS) / `var` は変更が必要な場合のみ
- **スコープの最小化**: 変数は使う場所の最も近いスコープで宣言する
- **マジックナンバーの排除**: 意味のある名前付き定数にする
- **一時変数の最小化**: map/filter/reduce のチェーンで中間変数を減らす

## 判断基準

- 再代入が必要か自問する。ほとんどの場合、関数型スタイルで不要にできる
- 変数名は役割を表す (`user` ではなく `admin`, `requestedBy` 等、文脈がある場合)
- 同じ値を複数箇所で使うなら名前付き定数にする (DRY)

## アンチパターン

- **Mutable Default**: 変更しないのに `let` / `var` で宣言
- **Wide Scope**: 関数先頭で全変数を宣言 (C 言語スタイル)
- **Magic Number**: `if (status === 3)` の `3` が何を意味するか不明
- **Accumulator Pattern**: `let result = []; for (...) { result.push(...) }` → `map/filter` で書ける

## 品質チェックポイント

- [ ] 変更しない変数が immutable で宣言されているか
- [ ] 変数のスコープが必要最小限か
- [ ] マジックナンバーが名前付き定数になっているか
- [ ] 再代入が関数型スタイルで置き換え可能でないか
