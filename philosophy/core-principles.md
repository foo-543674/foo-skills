# 基礎原則

すべての設計判断に適用する横断的な原則。個別の技術領域 (アーキテクチャ、API、データモデル等) の判断基準は perspectives/ に分離されているが、ここに書かれた原則はそのすべてに優先する。

## 1. 語彙定義義務

リポジトリで構造的な語彙 (層名・モジュール名・クラス名・ロール名) として使う単語は、その単語 1 つで何を意味するかが厳密に定義されていなければならない。**装飾語 (ドメイン名・形容詞) で範囲を狭めるのは定義ではない。**

### 判定テスト

その単語を「動詞 + 目的語」の責務として 1 文で書けるか。書けないなら構造的概念として使ってはいけない。

- `Validator` → 「入力と外部事実を受け取り、検証済み入力 or エラーを返す」 → OK
- `Service` → 「〜に関するロジックを集めたもの」(動詞がない) → NG
- `Manager` → 「〜を管理するもの」(管理 = 何でも入る) → NG

### 禁止語

サービス / マネージャ / ヘルパー / ユーティリティ / プロセッサ / ワーカー / エンジン / (確立した文脈以外の) ハンドラ。「ドメインサービス」「アプリケーションサービス」も同じ理由で構造的概念として使わない。代わりに具体的責務クラスの集合として表現する。

### なぜ必要か

曖昧な語を構造的概念として使うと、定義がないまま「なんとなく合うもの」を吸い込み続け、単一責任を保てなくなる。最終的に「変更理由が複数ある」「テストできない」「分割の指針がない」God オブジェクトに退化する。

## 2. 逃げの禁止と説明責任

「思いつかないからとりあえずこれ」「難しいから簡単な方に逃げる」を避ける。

### 典型的な逃げパターン

| パターン | 問題 | 代わりにすべきこと |
|---|---|---|
| **A. 曖昧な箱への退避** | 「サービス」「マネージャ」に放り込み、責務が言語化されない | 責務を「動詞+目的語」で言語化して具体的責務クラスを作る |
| **B. 数値的構造の文字列化** | 日付・金額・期間の比較・計算・並び替えが壊れる | 数値型 (DATE, DECIMAL 等) として扱う |
| **C. 直交次元の排他列挙化** | 組み合わせが増えるたびに列挙を拡張、設計的天井を作る | 直交する次元を独立した属性として表現する |
| **D. 安易な抽象化** | 1-2 回のパターンを「将来のため」に抽象化 | Rule of Three (3 回出るまで具象のまま) |
| **E. テストでの設計問題の吸収** | モック大量投入やテスト省略で設計の問題を隠す | テストの書きにくさは設計の問題のサイン。設計を直す |
| **F. 例外による検証エラー表現** | 「未検証/検証済み」を型で区別できなくなる | Result/Either で表現。例外はバグ専用 |

### 説明責任

逃げざるを得ない場合は、判断記録 (ADR / 設計ドキュメント) に以下を残す:
1. なぜそうせざるを得ないのか
2. 検討した代替案
3. 将来の見直しトリガー

この 3 点が書けない逃げは採用しない。

## 3. 捨てやすさ — 影響範囲のコントロール可能性

> **「捨てる時の影響範囲がコントロールできる状態か」**

技術的負債の本質は、外的要因 (ライブラリ廃止、ミドルウェア変更、フレームワーク陳腐化等) の変化が自分たちのコードにどこまで影響するかをコントロールできなくなった時に爆発する。

完全に隔離する必要はない。重要なのは「明日この技術がなくなったら何箇所修正が必要か」が **予測可能** で、**限定された場所** に閉じ込められていること。

### 4 つの隔離戦略

| 戦略 | 適する対象 | 例 |
|---|---|---|
| **A. ラッパー / 抽象化** | 内部詳細が露出しやすい技術基盤系 | DB クライアント、HTTP クライアント、ロガー、日付ライブラリ |
| **B. import 制限** | 中身が前面に出る、ラッパー化が困難 | UI コンポーネントライブラリ、状態管理ライブラリ |
| **C. レイヤー分離による結果的隔離** | IO / インフラに該当する | fs、ファイル IO、外部 API |
| **D. 並存可能な構造** | 一気に置換不可能な大型依存 | フレームワーク (段階的移行) |

### 過剰設計を避ける

- すべてを捨てやすくする必要はない
- 言語標準ライブラリは素のまま使う
- 薄い素通しラッパーは作らない
- 重要なのは「コントロール下にある」ことであって「完全に隔離されている」ことではない
